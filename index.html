Virtual Structure Approach for Multi-Robot Formation Control
See: GitHub & Simulation Video

Virtual structures consider the entire formation as a rigid body. The control law for a single robot is derived by defining the dynamics of the virtual structure and then translates the motion of the virtual structure into the desired motion of each robot. The basic algorithm iteratively fits the virtual structure to the robots positions via minimization of errors between them, moves virtual structure according to a local or global trajectory generator and updates the robot positions to follow the desired virtual structure trajectories. .

Custom build robot is design and manufactured. Arduino Mega has been used as lower level control for PID speed control and to read raw sensor data and to execute commands to actuators. Raspberry Pi used as middle level control to read IMU data and solve kinematics of robots. At top level Laptop is used for complex packages like robot_localization, imu_complementary_filter, OMPL (The Open Motion Planner Library). For indoor localization camera mounted on ceiling and Dewave Modules with Arduino Pro Mini is used.

Arduino code for waypoint tracking using GPS and Magnetometer.
See: GitHub & Simulation Video

Pre-defined Latitude and Longitude is tracked using GPS and Magnetometer, while avoiding the obstacles between the waypoints. GPS Data is used to find "Target Heading" and Magnetometer Placed on robot give us "Current Heading" of robot. And difference beteern "Target and Current Heading" will decide angular velocity of robot.
